# AI in Data Analytics — Northwind Project

Учебный проект в рамках курса **AI in Data Analytics**. Цель — научиться применять AI-инструменты (Claude Code) в процессе анализа данных и построить воспроизводимый аналитический проект: от сырых данных до выводов.

## Описание проекта

Проект посвящён анализу данных на примере классического учебного датасета Northwind. В рамках работы отрабатываются:

- построение воспроизводимого пайплайна анализа данных (Python + DuckDB + Pandas);
- работа с AI-ассистентом (Claude Code) как частью аналитического процесса — с фиксацией использованных промптов и явным разделением фактов из данных и AI-интерпретаций;
- базовые практики аналитической инженерии: разделение raw/processed данных, документирование допущений, ведение журнала работы.

Правила работы над проектом (в том числе для AI-ассистента) зафиксированы в [`CLAUDE.md`](./CLAUDE.md).

## Используемый датасет

**Northwind** — классический учебный датасет о вымышленной торговой компании (клиенты, сотрудники, заказы, товары, поставщики, территории продаж).

- **Формат:** CSV, отдельный файл на таблицу.
- **Источник:** [`neo4j-contrib/northwind-neo4j`](https://github.com/neo4j-contrib/northwind-neo4j).
- **Расположение в проекте:** `data/raw/` (исходные файлы, не изменяются вручную).
- **Состав таблиц и колонок:** подробное описание — в [`data/data_dictionary.md`](./data/data_dictionary.md).

Датасет закреплён окончательно; детальная валидация данных (проверка ключей, пропусков, ссылочной целостности) будет проведена на этапе анализа.

## Структура проекта

```
.
├── data/
│   ├── raw/                  # исходные CSV-файлы Northwind (не изменять вручную)
│   └── data_dictionary.md    # словарь данных: таблицы, колонки, ключи
├── notebooks/                # Jupyter-ноутбуки с исследовательским анализом
├── scripts/                  # вспомогательные и пайплайн-скрипты
├── src/                      # переиспользуемый код (загрузка, трансформации, утилиты)
├── prompts/                  # промпты, использованные при работе с AI
├── worklog/                  # журнал хода работы (даты, решения, обоснования)
├── CLAUDE.md                 # правила работы над проектом для AI-ассистента
├── VALIDATION_CHECKLIST.md   # чек-лист валидации данных/анализа
├── requirements.txt          # зависимости Python-окружения
└── README.md
```

## Используемые технологии

- **Python** — основной язык анализа;
- **Jupyter** — среда для исследовательского анализа данных;
- **DuckDB** — SQL-движок для запросов поверх CSV/датафреймов;
- **Pandas** — обработка и трансформация табличных данных;
- **Git / GitHub** — контроль версий и хранение проекта;
- **Claude Code** — AI-ассистент, используемый на всех этапах: от структуры проекта до анализа данных (с фиксацией промптов в `prompts/`).

## Как запустить проект

1. Клонировать репозиторий:
   ```bash
   git clone <URL-репозитория>
   cd AI-Analytics-Course
   ```
2. Создать и активировать виртуальное окружение:
   ```bash
   python -m venv .venv
   source .venv/bin/activate      # Windows: .venv\Scripts\activate
   ```
3. Установить зависимости:
   ```bash
   pip install -r requirements.txt
   ```
4. Запустить Jupyter:
   ```bash
   jupyter lab
   ```
5. Исходные данные уже находятся в `data/raw/` — дополнительная загрузка не требуется.
